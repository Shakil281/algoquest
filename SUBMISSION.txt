HOW TO RUN (30 seconds)
------------------------

No install. No build step. No dependencies to download.

Option 1 - Live site (instant):
  Open https://shakil281.github.io/algoquest/ in any modern browser.

Option 2 - Run locally:
  git clone https://github.com/Shakil281/algoquest.git
  cd algoquest
  Double-click index.html  (Windows / macOS / Linux)

That's it. One file. Works offline after first load (except the live trivia widget).


WHAT IS ALGOQUEST?
------------------

AlgoQuest is an interactive computer science learning game that teaches algorithms
through play — no coding knowledge required.

It turns abstract CS concepts (sorting, pathfinding, binary search, graph traversal)
into hands-on, visually engaging games with sound effects, 8-bit music, a mascot,
and a multiplayer mode.

The problem it solves:
CS education tools are either too dry (lecture slides, textbooks) or too abstract
(code editors). Students struggle to build an intuition for WHY algorithms work.
AlgoQuest bridges this gap by making algorithms something you feel and play,
not just read about.


THE FOUR GAMES
--------------

1. MONSTER SORTER — Sorting Algorithms
   - Drag and drop monsters (each with an HP value) from weakest to strongest
   - Three difficulty levels: Easy (6 monsters), Medium (8), Hard (10)
   - Play mode: drag-and-drop with a hint system (yellow glow shows optimal next swap)
   - Watch mode: watch the computer sort — choose Bubble Sort, Quick Sort, or Merge Sort
   - Live pseudocode panel highlights the exact line being executed at each step
   - Efficiency scoring: your move count vs. the theoretical minimum (inversions)
   - Combo system: consecutive optimal swaps multiply your score
   - Local multiplayer: up to 4 players take turns on the same device; ranked leaderboard

2. DUNGEON ESCAPE — Pathfinding (Dijkstra / A*)
   - 18x24 grid maze with draggable start (green) and finish (red) markers
   - Draw walls by clicking and dragging to reshape the maze
   - Run Dijkstra or A* and watch the search frontier expand cell by cell
   - Side-by-side comparison shows why A* visits far fewer cells (it uses a heuristic)
   - Queue/open-set panel updates in real time showing the algorithm's internal state
   - Step forward/backward controls + adjustable speed slider

3. NUMBER HUNT — Binary Search
   - 16 sorted numbers displayed as cards
   - Click any number to set it as the search target
   - Watch binary search halve the search range each step — eliminated numbers fade out
   - Left/Mid/Right pointer labels animate across the array
   - Panel explains: 16 numbers = at most 4 steps. 1,000,000 numbers = at most 20 steps.

4. MAP EXPLORER — BFS / DFS
   - 10-node graph representing a map of connected places (A through J)
   - Click any node to set the start, then run BFS or DFS
   - BFS spreads outward level-by-level (ripple); DFS dives deep before backtracking
   - Traversal order badge row builds up in real time
   - Queue (BFS) / Stack (DFS) panel shows the algorithm's internal data structure live


API INTEGRATION — Open Trivia Database
---------------------------------------

Endpoint: https://opentdb.com/api.php?amount=1&category=18&type=multiple
(Category 18 = Science: Computers. Free, no API key required.)

A CS Trivia widget sits on the main menu and:
  - Fetches a fresh Computer Science multiple-choice question on every visit
  - Shuffles the 4 answer options randomly
  - Shows difficulty (easy / medium / hard) and the exact sub-category
  - Highlights correct answers in green, wrong answers in red after selection
  - Tracks a streak counter persisted in localStorage
  - Reacts with the PINGU mascot (great animation on correct, sad on wrong)
  - Has a spinner loading state and a retry button on network failure
  - "NEXT" button fetches the next question immediately without a page reload

TECHNICAL ARCHITECTURE
-----------------------

Everything lives in a single index.html file (~1,950 lines).

  UI framework      React 18 (loaded from CDN via UMD build)
  JSX compilation   Babel Standalone (in-browser transform)
  Fonts             Google Fonts — Press Start 2P (pixel font)
  Audio             Web Audio API — custom oscillator-based sound engine
  Music             Lookahead scheduler pattern — 8-bar chiptune loop
  Persistence       localStorage (scores, theme, planet, mute, trivia streak)
  External API      Open Trivia Database (opentdb.com)
  Deployment        GitHub Pages (static, zero-config)

No backend. No database. No authentication. No node_modules.

Algorithm implementations (all written from scratch):
  - Bubble Sort    generates full step trace with comparing/swapping/sorted state
  - Quick Sort     Lomuto partition scheme with pivot highlighting
  - Merge Sort     divide-and-conquer with merge-region highlighting
  - Dijkstra / A*  on an 18x24 grid with wall obstacles, produces animation frames
  - Binary Search  step-by-step with lo/mid/hi pointer state per frame
  - BFS            queue-based level-order traversal on a 10-node graph
  - DFS            stack-based depth-first traversal on the same graph


SCORING SYSTEM
--------------

Monster Sorter uses inversion count to compute the theoretical minimum swaps:
  - Count inversions in the initial array = optimal swap count
  - Ratio of player swaps to optimal swaps determines star rating:
      <= 1.3x optimal  =>  3 stars
      <= 2.0x optimal  =>  2 stars
      Otherwise        =>  1 star
  - Best scores persist per level
  - Medium unlocks after 1 star on Easy; Hard unlocks after 1 star on Medium
